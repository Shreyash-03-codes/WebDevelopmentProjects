Insertion Sort:

        Definition: Insertion Sort is a simple and intuitive sorting
        algorithm that builds the final sorted array one item at a time. It takes 
        elements from the unsorted part of the array and inserts them into their 
        correct position in the sorted part, shifting larger elements to the right if 
        necessary.

Intuition of Insertion Sort:

    Intuition:
        Imagine sorting a deck of cards in your hands. You pick up cards one by one 
        and place each card in the correct position relative to the ones you've 
        already sorted. Each new card is compared with the already sorted cards, 
        starting from the right, and inserted at the appropriate position. Insertion 
        sort behaves similarly, where the algorithm divides the array into a "sorted" 
        part and an "unsorted" part, and it gradually builds the sorted array by 
        inserting each element into its correct position.

Here is the Java code for Insertion Sort along with an example: 

            import java.util.Arrays;

            public class InsertionSortExample {

                // Method to perform insertion sort
                public static void insertionSort(int[] array) {
                    for (int i = 1; i < array.length; i++) {
                        int key = array[i];
                        int j = i - 1;
                        
                        // Shift elements of the sorted section to the right
                        while (j >= 0 && array[j] > key) {
                            array[j + 1] = array[j];
                            j = j - 1;
                        }
                        // Insert the key at its correct position
                        array[j + 1] = key;
                    }
                }

                public static void main(String[] args) {
                    // Example input array
                    int[] array = {7, 3, 5, 1};
                    
                    System.out.println("Original array: " + Arrays.toString(array));
                    
                    // Call insertion sort method
                    insertionSort(array);
                    
                    // Output sorted array
                    System.out.println("Sorted array: " + Arrays.toString(array));
                }
            }

        

            Let's step through the iterations for the array [7, 3, 5, 1].

            Initial Array:

            [7, 3, 5, 1]
            Iteration 1:
            The first element 7 is considered sorted, so we move to the second element 3.
            Compare 3 with 7. Since 3 < 7, shift 7 to the right and insert 3 at index 0.
           
            [3, 7, 5, 1]

            Array after 1st iteration:

            [3, 7, 5, 1]

            Iteration 2:
            Move to the third element 5.
            Compare 5 with 7. Since 5 < 7, shift 7 to the right.
            Compare 5 with 3. Since 5 > 3, insert 5 at index 1.
           
            [3, 5, 7, 1]

            Array after 2nd iteration:

            [3, 5, 7, 1]

            Iteration 3:
            Move to the fourth element 1.
            Compare 1 with 7. Since 1 < 7, shift 7 to the right.
            Compare 1 with 5. Since 1 < 5, shift 5 to the right.
            Compare 1 with 3. Since 1 < 3, shift 3 to the right.
            Insert 1 at index 0.
            
            [1, 3, 5, 7]
            Array after 3rd iteration:

            [1, 3, 5, 7]

            The array is now fully sorted after 3 iterations.

Time Complexity:


            Best Case (when array is already sorted): O(n)

            Average Case:O(n²)

            Worst Case: O(n²)

            The time complexity is quadratic for the average and worst case
            because for each element, insertion sort may have to shift a number 
            of elements to the right, resulting in nested loops.

Space Complexity:

            Space Complexity: O(1) (In-place sorting algorithm)
            Bubble sort requires a constant amount of extra memory space, as it modifies the array directly.

Advantages:

  1. Simple and Intuitive: Easy to understand and implement
  2. Efficient for Small Data Sets: Works efficiently with small arrays or lists.
  3. Adaptive: It is efficient for data that is already sorted or nearly sorted (O(n) for best case).
  4. Stable: Does not change the relative order of elements with equal keys.

Disadvantages:

  1. Inefficient for Large Data Sets: Time complexity of O(n²) makes it slow for large lists.
  2. Many Shifts: In the worst case, a large number of shifts are needed to insert an element in the correct position.

Applications:

  1. Small Data Sets: It works well for small datasets or partially sorted datasets.
  2. Online Sorting: Insertion sort can be useful when the data arrives in a stream, and we want to keep the list sorted as new elements are added.
  3. Almost Sorted Data: Works efficiently when the data is almost sorted (adaptive).
Note:The Insertion sort algorithm sorts the array step-by-step, inserting each element into its correct position in the sorted part of the array.
