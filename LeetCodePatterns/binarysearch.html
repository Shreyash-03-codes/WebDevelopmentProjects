Binary Search

Definition: Binary Search is a search algorithm that finds the position of a target element within a sorted array or list by repeatedly dividing the search interval in half. If the target value is equal to the middle element, the search ends. If the target is less than the middle element, the search continues in the left half; if greater, it continues in the right half.

Intuition: The intuition behind binary search is that in a sorted list, you can ignore half of the elements in each step, making it highly efficient for large datasets. Since every step reduces the search space by half, it quickly converges to the target, if it exists.

    Required Data Structures:

  • Array or List:The input data must be stored in a sorted array or list for binary search to be applicable..

    Algorithm:

  1. Initialization:
    • Two pointers (left and right) are set to the beginning and end of the array.
  2. Binary Search Loop:
    • Calculate the middle index mid as left + (right - left) / 2.
    • If arr[mid] equals the target, return mid (target found).
    • If arr[mid] is less than the target, move the left pointer to mid + 1 (search in the right half).
  3. Return -1 if Not Found:
    • If the loop exits without finding the target, return -1 (target not present in the array).

Java Code Example:

Below is an example of binary search implemented iteratively in Java.

        public class BinarySearchExample {
            // Method to perform binary search
            public static int binarySearch(int[] arr, int target) {
                int left = 0;
                int right = arr.length - 1;
        
                while (left <= right) {
                    int mid = left + (right - left) / 2;
        
                    // Check if the target is present at mid
                    if (arr[mid] == target) {
                        return mid; // Target found at index mid
                    }
        
                    // If target is greater, ignore the left half
                    if (arr[mid] < target) {
                        left = mid + 1;
                    }
                    // If target is smaller, ignore the right half
                    else {
                        right = mid - 1;
                    }
                }
        
                // Target not found in the array
                return -1;
            }
        
            public static void main(String[] args) {
                int[] arr = {1, 3, 5, 7, 9, 11};
                int target = 7;
                int result = binarySearch(arr, target);
        
                if (result == -1) {
                    System.out.println("Element not found in the array");
                } else {
                    System.out.println("Element found at index: " + result);
                }
            }
        }

Input:

arr = {1, 3, 5, 7, 9, 11} target = 7

Output:

Element found at index: 3

    Execution Steps:

  1. Start with left = 0 and right = 5 (the length of arr minus one).
  2. Calculate mid = 2 (the element at index 2 is 5).
  3. Since 5 < 7, move left to mid + 1 = 3.
  4. Recalculate mid = 4 (the element at index 4 is 9).
  5. Since 9 > 7, move right to mid - 1 = 3.
  6. Recalculate mid = 3 (the element at index 3 is 7).
  7. Since arr[mid] == target, return mid, which is 3.

    Time Complexity:

  • Time Complexity: O(log n) because each comparison reduces the search space by half.

    Space Complexity:

  • Space Complexity: O(1) for iterative binary search (no additional data structure needed), and O(log n) for recursive binary search (due to recursion stack)..

    Advantages:

  1. Efficiency: Binary search has a logarithmic time complexity of O(log n), making it efficient for large, sorted datasets
  2. Memory Usage: The iterative version has O(1) space complexity, requiring no additional memory beyond the input array

    Disadvantages:

  1. Sorted Data Requirement: Binary search requires a sorted array. Sorting the data before performing binary search adds an O(n log n) cost.
  2. Not Suitable for Linked Lists: Binary search is less efficient for linked lists because accessing the middle element in a linked list is O(n).

    Applications:

  1. Finding Elements in Sorted Data:: Quickly find elements or confirm the absence of elements in sorted arrays
  2. Search Problems in Coding Competitions: Frequently used for solving search-related problems.
  3. Efficiently Solving Mathematical Problems: Binary search is used in numerical methods to find roots or optimize functions.
  4. Finding Peaks and Boundaries: Used in peak-finding algorithms or boundary condition checks in array-based problems.
