Greedy

Definition: A Greedy Algorithm is an approach that builds a solution by repeatedly choosing the best option available at each step, without revisiting past choices. Greedy algorithms work well when the problem has an "optimal substructure" and no need for backtracking or future revisions.

Intuition: The intuition behind greedy algorithms is to make choices that seem the best in the short term, with the assumption that they will lead to an optimal solution in the end. This is particularly effective when a problem can be broken down into subproblems with an optimal substructure, meaning that each local optimum decision contributes to the global optimum.

    Required Data Structures:

    • Arrays or Lists: For storing items or elements to process (often sorted for easier access).
    • Priority Queue or Heap: Useful in cases where the maximum or minimum item needs to be selected repeatedly.

    Algorithm:

    1. Initialize the Solution:
      • Start with an initial solution or set of parameters to track the solution as you progress.
    2. Make Greedy Choices:
      • At each step, pick the option that offers the most benefit or is locally optimal based on the problem’s requirements.
    3. Update the Solution:
      • Incorporate the selected choice into the current solution state.
    4. Repeat Until Completion:
      • Continue making greedy choices until you reach the end or the solution is fully constructed.
    5. Return the Result:
      • Return the solution once all choices have been made and integrated.

Java Code Example:

Problem: Given a set of activities with start and end times, select the maximum number of activities that do not overlap. This is known as the Activity Selection Problem.

        import java.util.Arrays;
        import java.util.Comparator;
        
        public class ActivitySelection {
            public static int maxActivities(int[][] activities) {
                // Sort activities based on their end times
                Arrays.sort(activities, Comparator.comparingInt(a -> a[1]));
        
                int count = 1; // Select the first activity
                int end = activities[0][1]; // Track the end time of the last selected activity
        
                for (int i = 1; i < activities.length; i++) {
                    // If the start time of the current activity is after or at the end time of the last selected activity
                    if (activities[i][0] >= end) {
                        count++; // Select this activity
                        end = activities[i][1]; // Update end time to the end of the current activity
                    }
                }
                return count;
            }
        }

Input:

Activities: [[1, 3], [2, 5], [4, 6], [6, 8], [5, 7]]

Output:

Maximum number of activities: 3 (e.g., activities [1, 3], [4, 6], and [6, 8]).

    Execution Steps:

    1. Sort Activities by End Time: Select the first activity (the earliest finishing one) and initialize the end time to its end.
      • Sort the array of activities by their end time to facilitate picking the earliest finishing activity that doesn’t overlap
    2. Initialize Counters and Track the End Time:
      • Select the first activity (the earliest finishing one) and initialize the end time to its end.
    3. Iterate Through Activities:
      • For each subsequent activity, check if its start time is greater than or equal to the current end time
      • If so, include it in the solution, increment the count, and update the end time to the end of the newly selected activity.
    4. Return the Result:
      1. After processing all activities, return the count of selected activities

    Example Execution for Input [[1, 3], [2, 5], [4, 6], [6, 8], [5, 7]]

    • Sorted Activities: [[1, 3], [4, 6], [5, 7], [6, 8], [2, 5]]
    • Select [1, 3], then [4, 6], and finally [6, 8].
    • Result: 3 activities can be selected without overlap.

    Time Complexity:

    • Greedy algorithms typically run in O(NlogN) or O(N), depending on whether sorting is required or if a single pass is sufficient.
    • Knapsack Problem with Greedy: Sorting the items by value-to-weight ratio is O(NlogN), and selecting items is O(N), resulting in O(NlogN).

    Space Complexity:

  • Most greedy algorithms require O(1) or O(N) space, mainly for storing interim results or processing items sequentially.

    Advantages:

    1. Efficient and Fast: Greedy algorithms generally have low time complexity (often O(N) or O(NlogN)), making them efficient.
    2. Simple and Easy to Implement: The step-by-step selection process is straightforward, often requiring fewer data structures than other approaches.
    3. Optimal Solutions for Specific Problem Types: For certain problems, like minimum spanning tree, greedy provides an optimal solution

    Disadvantages:

    1. Non-Optimal Solutions for All Problems: Greedy algorithms may not provide the optimal solution for problems where local choices don’t lead to a global optimum (e.g., 0/1 knapsack problem).
    2. Limited Applicability: Only works for problems with optimal substructure and no backtracking requirements.
    3. No Global Perspective: Greedy algorithms focus only on local choices, which can lead to suboptimal solutions for some problems.

    Applications:

    1. Optimization Problems: Problems like minimum spanning tree (Kruskal’s and Prim’s algorithms), Dijkstra's shortest path, and Huffman encoding.
    2. Scheduling Problems: Activity selection, job scheduling with deadlines.
    3. Knapsack Problem (Fractional): The fractional knapsack problem, where items can be broken into smaller parts, is solvable optimally with greedy
    4. Graph Problems: Greedy is used in graph algorithms for minimum spanning tree, shortest paths, and approximation algorithms.
    5. Cashier Problem: Finding the minimum number of coins or bills for a given amount.
    6. Resource Allocation: Problems requiring efficient allocation of resources, like interval scheduling or task assignment.
