sliding window

Definition: The sliding window approach involves maintaining a "window" of data, moving it across the dataset while keeping track of relevant information. The window can be fixed-size (where the window length remains constant) or variable-size (where the window expands and contracts based on conditions).

For example, to find the maximum sum of a subarray of length k in an array, a sliding window approach can move a window of size k across the array, calculating sums efficiently by reusing the results from the previous window position.

Intuition: The intuition behind the sliding window algorithm is to use a small "window" to look at only a portion of data at any given time, moving this window across the dataset to gather insights or perform calculations. Instead of recalculating data for each new segment of the dataset, you reuse information from the previous window, which saves time and makes the algorithm efficient.

    Required Data Structures:

  • Arrays or Lists: Sliding window is often applied to arrays or lists, where each element is part of the window.
  • Deques: For problems involving maximum or minimum in each window, a deque can be used to keep track of relevant indices in the window efficiently.
  • Heaps: When dynamically maintaining a sorted order of elements in the window, heaps may be used, though they increase complexity to O(logk).

    Algorithm:

  1. Initialize the Window:
    • Calculate the sum of the first k elements in the array and store this as the initial window sum.
  2. Slide the Window Across the Array:
    • Move the window one element to the right, by:
      • Subtracting the element that’s leaving the window (the leftmost element of the previous window).
      • Adding the new element that’s entering the window (the next element in the array).
    • Update the maximum sum if the new window sum is larger than the previously stored maximum.
  3. Continue Until the End:
    • Repeat the sliding operation until you reach the end of the array.
  4. Result:
    • The stored maximum sum at the end of the loop is the answer.

Java Code Example:

Here’s a Java code example for finding the maximum sum of a subarray of fixed length k using a sliding window.

    public class SlidingWindowExample {
        public static int maxSumSubarray(int[] arr, int k) {
            if (arr.length < k) {
                throw new IllegalArgumentException("Array length must be at least k.");
            }
    
            int maxSum = 0;
            int windowSum = 0;
    
            // Calculate initial window sum
            for (int i = 0; i < k; i++) {
                windowSum += arr[i];
            }
    
            maxSum = windowSum;
    
            // Slide the window
            for (int i = k; i < arr.length; i++) {
                windowSum += arr[i] - arr[i - k]; // add the new element and remove the left element
                maxSum = Math.max(maxSum, windowSum);
            }
    
            return maxSum;
        }
    
        public static void main(String[] args) {
            int[] arr = {1, 2, 3, 4, 5, 6, 7, 8};
            int k = 3;
            System.out.println("Maximum sum of a subarray of length " + k + " is: " + maxSumSubarray(arr, k));
        }
    }

Input:

[1, 2, 3, 4, 5, 6, 7, 8] k=3

Output:

Maximum sum of a subarray of length 3 is: 21

    Execution Steps:

  1. Step 1: Calculate the Initial Window Sum for the First k=3 Elements
    • Window: [1, 2, 3]
      • windowSum = 1 + 2 + 3 = 6
      • Set maxSum = 6
  2. Step 2: Slide the Window Across the Array and Update the Maximum Sum

    Now, we slide the window one element at a time, removing the element that is going out of the window (leftmost element) and adding the new element that comes into the window (rightmost element).

    1. Move to index i = 3:
      • New Window: [2, 3, 4]
      • Update windowSum: windowSum = 6 - 1 + 4 = 9
      • Update maxSum: maxSum = Math.max(6, 9) = 9
    2. Move to index i = 4:
      • New Window: [3, 4, 5]
      • Update windowSum: windowSum = 9 - 2 + 5 = 12
      • Update maxSum: maxSum = Math.max(9, 12) = 12
    3. Move to index i = 5:
      • New Window: [4, 5, 6]
      • Update windowSum: windowSum = 12 - 3 + 6 = 15
      • Update maxSum: maxSum = Math.max(12, 15) = 15
    4. Move to index i = 6:
      • New Window: [5, 6, 7]
      • Update windowSum: windowSum = 15 - 4 + 7 = 18
      • Update maxSum: maxSum = Math.max(15, 18) = 18
    5. Move to index i = 7:
      • New Window: [6, 7, 8]
      • Update windowSum: windowSum = 18 - 5 + 8 = 21
      • Update maxSum: maxSum = Math.max(18, 21) = 21
  3. Step 3: End of Array
    • The loop completes as we’ve reached the end of the array.
    • The final maxSum is 21, which is the maximum sum of any subarray of length 3 in [1, 2, 3, 4, 5, 6, 7, 8].

    Time Complexity:

    • Time Complexity: O(n)
      • The algorithm typically processes each element once, resulting in linear time complexity.

    Space Complexity:

    • Space Complexity: O(1) or O(k)
      • For a fixed-size sliding window, space complexity is O(1) if we only need to track a few values, or O(k) if the window's elements are stored directly.

    Advantages:

    • Efficient for Large Arrays: Only requires a single traversal of the array, making it time-efficient for large datasets.
    • Adaptable to Many Problems: Useful in problems involving subarrays, substrings, or other sequential data
    • Simple to Implement: The logic is straightforward for many standard problems, and implementations are usually easy to understand.

    Disadvantages:

    • Limited to Sequential Data: Best suited for problems where a contiguous segment of data is required.
    • Not Suitable for Dynamic Windows with Arbitrary Constraints: If the window size must change frequently or if conditions are complex, it can be challenging to implement effectively.

    Applications:

    1. Fixed-Length Subarray Problems: Finding the maximum or minimum sum/product of a subarray of fixed length.
    2. Variable-Length Subarray Problems: Finding subarrays that meet a certain condition (e.g., subarrays with a target sum).
    3. String Problems: Finding the longest substring with specific properties (e.g., unique characters, specific character counts).
    4. Real-Time Data Processing: Often used in systems that need to analyze real-time data, such as moving averages in time series data or sensor readings.
    5. Network Data Analysis: Used in network traffic analysis to detect anomalies within a moving time window of data packets.
