Two Pointers

Definition: The two pointers technique is an algorithmic approach that involves using two pointers (or indices) to traverse a data structure in a specific manner. The pointers can start at various positions, such as from the beginning and end of an array, or both from the start, depending on the problem requirements. This technique is useful in reducing time complexity for problems that involve searching for pairs, ranges, or specific patterns within ordered structures

Intuition: The two pointers technique takes advantage of sorted or semi-structured data, allowing one pointer to increase and the other to decrease, or both to progress, based on a condition. For example, in an array sorted in ascending order, two pointers moving towards each other from the ends can quickly locate elements that sum to a target value without needing nested loops.

    Required Data Structures:

  • Arrays or Lists:Ideal for problems involving sequential elements.
  • Strings: Commonly used in problems like palindrome checks.
  • Linked Lists:Useful for problems like cycle detection in linked lists using Floyd’s Tortoise and Hare algorithm.

    Algorithm:

  1. Initialization: Two pointers (left and right) are set to the beginning and end of the array.
  2. Loop: The pointers move towards each other. The sum of the elements at these pointers is calculated.
  3. Condition Check:
    • If the sum equals the target, the method returns true.
    • If the sum is less than the target, the left pointer increments (moves to a larger value).
    • If the sum is greater than the target, the right pointer decrements (moves to a smaller value).
  4. Termination: If the pointers cross without finding a matching sum, the method returns false.

Java Code Example:

Below is an example using the two pointers technique to check if a sorted array has two numbers that add up to a target sum

    public class TwoPointersExample {
        public static boolean hasPairWithSum(int[] arr, int target) {
            int left = 0;
            int right = arr.length - 1;

            while (left < right) {
                int sum = arr[left] + arr[right];
                if (sum == target) {
                    return true;  // Found a pair with the target sum
                } else if (sum < target) {
                    left++;  // Increase the left pointer to get a larger sum
                } else {
                    right--;  // Decrease the right pointer to get a smaller sum
                }
            }
            return false;  // No pair found with the target sum
        }

        public static void main(String[] args) {
            int[] arr = {1, 2, 4, 4, 6, 10};
            int target = 8;
            System.out.println("Has Pair with Sum: " + hasPairWithSum(arr, target));
        }
    }

Input:

arr = {1, 2, 4, 4, 6, 10} target = 8

Output:

Has Pair with Sum: true

    Execution Steps:

  1. Start with left pointer at index 0 (value 1) and right pointer at index 5 (value 10).
  2. Sum = arr[left] + arr[right] = 1 + 10 = 11 (greater than 8), so move the right pointer to the left (to index 4).
  3. New sum = arr[left] + arr[right] = 1 + 6 = 7 (less than 8), so move the left pointer to the right (to index 1).
  4. New sum = arr[left] + arr[right] = 2 + 6 = 8 (equals the target), so return true.

    Time Complexity:

    For most problems, the time complexity of the two pointers technique is:

  • O(n), where n is the number of elements in the data structure. This efficiency is due to the fact that each pointer usually only passes through the data structure once.

    Space Complexity:

  • O(1), as it typically only requires a constant amount of extra space for the pointers and possibly a few other variables.

    Advantages:

  1. Efficient: Reduces the need for nested loops, often cutting down runtime from O(N2) to O(N).
  2. Simple: Allows intuitive solutions with straightforward logic for common problems like finding pairs, partitions, or reversing subarrays.

    Disadvantages:

  1. Limited Use: Works mainly on problems with ordered structures or problems that can be broken into two halves.
  2. Restricted to Certain Patterns: Not applicable for all problem types, especially those without a clear two-pointer structure.

    Applications:

  1. Two Sum Problem: Find two elements that sum up to a target
  2. Palindrome Check: Verify if a string reads the same backward and forward.
  3. Cycle Detection: Detect loops in linked lists using Floyd’s Tortoise and Hare algorithm.
  4. Partitioning: Separate even and odd numbers, positive and negative numbers, etc.
  5. Sorting: Used in the quicksort partitioning phase.
