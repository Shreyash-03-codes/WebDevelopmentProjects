Prefix Sum Technique

Definition: The prefix sum technique involves creating an array (or list) where each element at index i represents the sum of all elements in the original array from the start up to that index. This technique is useful for efficiently calculating the sum of elements over any contiguous subarray, as it avoids recalculating sums repeatedly.

Intuition: The intuition behind the prefix sum technique is that once we have the sum of elements up to each index in an array, we can calculate the sum of any subarray quickly. For example, to find the sum from index left to right, we can subtract the prefix sum at left−1 from the prefix sum at right. This eliminates the need to recompute sums for overlapping subarrays repeatedly, saving time in range-sum problems.

    Required Data Structures:

  • Arrays or Lists:Array or List: Used for storing the prefix sums, enabling fast access to cumulative sums.

    Algorithm:

  1. Building the Prefix Sum Array:
    • The buildPrefixSum method constructs the prefix sum array by iterating over the original array.
    • Each entry at index i in prefixSum contains the cumulative sum from the start of the array to index i.
  2. Querying the Range Sum:
    • The getRangeSum method calculates the sum of elements between two indices (left and right) using the prefix sum array.
    • If left is 0, it directly returns prefixSum[right].
    • Otherwise, it subtracts prefixSum[left - 1] from prefixSum[right] to obtain the sum for the range.

Java Code Example:

The following example uses the prefix sum technique to find the sum of elements between any two indices in an array.

        public class PrefixSumExample {
            // Method to build prefix sum array
            public static int[] buildPrefixSum(int[] arr) {
                int[] prefixSum = new int[arr.length];
                prefixSum[0] = arr[0];
        
                for (int i = 1; i < arr.length; i++) {
                    prefixSum[i] = prefixSum[i - 1] + arr[i];
                }
        
                return prefixSum;
            }
        
            // Method to get the sum of elements between indices left and right
            public static int getRangeSum(int[] prefixSum, int left, int right) {
                if (left == 0) {
                    return prefixSum[right];
                }
                return prefixSum[right] - prefixSum[left - 1];
            }
        
            public static void main(String[] args) {
                int[] arr = {3, 1, 4, 1, 5, 9, 2};
                int[] prefixSum = buildPrefixSum(arr);
                
                // Example query: sum of elements from index 1 to index 4
                int left = 1;
                int right = 4;
                int sum = getRangeSum(prefixSum, left, right);
                System.out.println("Sum of elements from index " + left + " to " + right + " is: " + sum);
            }
        }

Input:

arr = {3, 1, 4, 1, 5, 9, 2}
Query for the sum from index 1 to 4.

Output:

Sum of elements from index 1 to 4 is: 11

    Execution Steps:

  1. Initialization:
    • prefixSum[0] = arr[0] = 3 So, the prefix sum array starts as {3}.
  2. Loop Iterations:
    • Iteration 1:i=1 prefixSum[1]=prefixSum[0]+arr[1]=3+1=4 The prefix sum array is now {3, 4}.
    • Iteration 2:i=2 prefixSum[2]=prefixSum[1]+arr[2]=4+4=8 The prefix sum array becomes {3, 4, 8}.
    • Iteration 3: i=3 prefixSum[3]=prefixSum[2]+arr[3]=8+1=9 The prefix sum array updates to {3, 4, 8, 9}.
    • Iteration 4: i=4 prefixSum[4]=prefixSum[3]+arr[4]=9+5=14 The prefix sum array updates to {3, 4, 8, 9, 14}.
    • Iteration 5:i=5 prefixSum[5]=prefixSum[4]+arr[5]=14+9=23 The prefix sum array updates to {3, 4, 8, 9, 14, 23}.
    • Iteration 6:i=6 prefixSum[6]=prefixSum[5]+arr[6]=23+2=25 The prefix sum array updates to {3, 4, 8, 9, 14, 23, 25}.
  3. Final Prefix Sum Array:After all iterations, the prefix sum array is {3, 4, 8, 9, 14, 23, 25}.
  4. Inputs: prefixSum = {3, 4, 8, 9, 14, 23, 25}, left = 1, and right = 4.
  5. Case Check:Since left is not 0, we use the formula: Range Sum=prefixSum[4]−prefixSum[1−1]
  6. Calculations
    • prefixSum[4] = 14 (sum of elements from index 0 to 4 in arr).
    • prefixSum[0] = 3 (sum of elements from index 0 to 0 in arr).
    • Range Sum = 14 - 3 = 11

    Time Complexity:

  • Constructing the prefix sum array takes O(n) time, where n is the length of the original array.
  • Each range-sum query takes O(1) time, making it efficient for multiple queries.

    Space Complexity:

  • O(n), as an additional array of the same length as the original array is required for storing the prefix sums.

    Advantages:

  1. Efficiency for Multiple Queries: Once the prefix sum array is built, any range sum query can be answered in constant time O(1), making it ideal for applications requiring multiple queries.
  2. Reduced Redundancy: Avoids recalculating the sum for overlapping elements in subarrays, saving computational effort.

    Disadvantages:

  1. Extra Space: Requires additional storage space for the prefix sum array, which may be a drawback for memory-constrained systems.
  2. Static Data: The prefix sum array needs to be recomputed if the original array is modified, making it less efficient for applications with frequent data updates..

    Applications:

  1. Range Sum Queries: Efficiently handles range sum queries in competitive programming and database applications.
  2. Subarray Sum Problems: Used to solve problems involving subarray sums, such as finding the maximum subarray sum.
  3. Frequency Count Problems: Helps in problems involving counting occurrences within specific ranges.
  4. Cumulative Analysis: Useful for cumulative calculations over ordered data, such as running totals, weather data, or financial analytics.
