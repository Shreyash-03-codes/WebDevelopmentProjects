Depth First Search (DFS) of tree

Definition: DFS is a recursive or iterative traversal technique where each node is visited by fully exploring one branch of the tree before moving to another. It uses a Last In, First Out (LIFO) data structure (a stack) to manage traversal, either explicitly in an iterative approach or implicitly through the call stack in recursion.

Intuition: The intuition behind DFS is to dive deep into the tree, exploring each child of a node before backtracking. Starting from the root, DFS goes as far down as possible along a branch, visiting nodes deeply before returning to higher levels. This allows complete exploration of one path at a time.

    Required Data Structures:

    • Stack: For iterative DFS, a stack is used to track nodes to be visited
    • Recursion Stack: In recursive DFS, the system call stack manages the nodes, but no explicit stack is required.

    Algorithm:

    1. Choose Starting Node:
      • Start from the root node of the tree.
    2. Visit Node:
      • Mark the current node as visited or process its data as needed.
    3. Traverse Children:
      • For each child of the current node:
        • Recursively apply DFS if using recursion, or push it onto the stack if using iteration.
    4. Backtrack When Needed:
      • Once all descendants of a node are visited, backtrack to the parent node, either by recursion return or by popping the stack.
    5. Complete Traversal:
      • Continue until all nodes are visited.

Java Code Example:

Problem: Perform DFS on a binary tree and print nodes in pre-order (root, left, right). 

        class TreeNode {
            int val;
            TreeNode left;
            TreeNode right;
            TreeNode(int x) { val = x; }
        }
        
        public class TreeDFS {
            public void dfs(TreeNode root) {
                if (root == null) return;
                
                // Process the current node
                System.out.print(root.val + " ");
                
                // Recursive calls for left and right subtrees
                dfs(root.left);
                dfs(root.right);
            }
        }

Input:

tree structure:

                         1                       
                        / \
                       2   3
                      / \   \
                     4   5   6

Output:

Result: 1 2 4 5 3 6 (Pre-order traversal).

    Execution Steps:

    1. Start from Root:
      • Begin DFS from the root node 1.
    2. Visit Nodes in Pre-order:
      • Visit the root node 1 and print its value.
      • Recursively move to the left child, visiting 2, then 4, backtracking after reaching the leaf node.
    3. Backtrack and Continue:
      • After visiting left descendants of 2, visit its right child 5 and backtrack to 1.
      • Move to the right subtree, visiting 3, then 6, completing the traversal.

    Example Execution for Tree Structure Above

    • Visit 1
    • Visit 2
    • Visit 4 (left leaf of 2)
    • Backtrack to 2, visit 5 (right child of 2)
    • Backtrack to 1, visit 3
    • Move to 3’s right child, visit 6
    • Traversal complete

    Final result: 1 2 4 5 3 6.

    Time Complexity:

  • DFS of Tree: O(N), where N is the number of nodes in the tree, as each node is visited once

    Space Complexity:

    • Recursive DFS: O(H) due to the recursion stack, where H is the height of the tree.
    • Iterative DFS: O(H), where a stack is used to manage the traversal order, consuming space based on tree height

    Advantages:

    1. Simple to Implement: DFS is straightforward to code with recursion.
    2. Suitable for Pathfinding: DFS efficiently finds paths from root to any other node.
    3. Useful for Tree Analysis: DFS can identify nodes in various traversal orders (pre-order, in-order, post-order) useful for specific applications

    Disadvantages:

    1. Stack Overflow for Large Trees: Recursive DFS can lead to stack overflow for deep trees.
    2. May Not Find Shortest Path: Unlike BFS, DFS does not guarantee the shortest path in trees or graphs.
    3. Complex Backtracking for Some Problems:Complex Backtracking for Some Problems: DFS may not be ideal for problems requiring backtracking at multiple levels.

    Applications:

    1. Tree Traversal: DFS is the standard approach for traversing tree structures in pre-order, in-order, and post-order.
    2. Pathfinding: DFS finds paths in binary trees or graphs, although it’s less suited for shortest path problems.
    3. Connected Components: DFS can identify all nodes connected to a given node in a tree or graph.
    4. Cycle Detection: In directed graphs, DFS can help identify cycles.
    5. Topological Sorting: DFS-based algorithms can determine task scheduling in directed acyclic graphs (DAGs)
    6. Puzzle Solving: Problems like solving mazes or finding sequences use DFS for exploration.
